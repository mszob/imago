# key layout

My Planck layout is thought-out as a sort of finger-extention system. This involves both hardware decisions — e.g. size and placement of the spacebar — and software decisions — e.g. number and placement of layer keys.

Layer 1 is broken down below.

# finger placement

  • primary key: finger rests here
  • secondary key: within one key of finger resting position
  • palm key: underside of piky knuckle hovers over and strikes this key
  • reach key: smaller keyboards like the Corne omit these. They are used for things like media control, not primary alphanumeric functions.

# key value

The same key may trigger a tap function and a hold function

  • prime tap: primary and seconday keys with alphanumeric function on tap
  • palm tap: palm key with backspace or enter a on tap
  • reach tap: reach keys with lesser functions on tap
  • mod hold: system mod (Win;Cmd), Opt, Ctrl, or Shift on hold
  • layer hold: layer toggle on hold
  • macro hold: custom macro on hold

# layer breakdown

  • VL: volume
  • BR: brightness
  • RC: macro record
  • RP: macro replay
  • L1: layer 1 toggle
